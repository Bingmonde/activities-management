Le Bootstrap est une libraire pour faciliter le design de page web. Un fonctionnement très logique et puissant de Bootstrap est qu’il utilise le système de grilles. Les utilisateurs peuvent créer les lignes et les cols selon leurs besoins pour gérer la vue de la page.
Lorsque la taille de fenêtre change, il est possible que certains blocs doivent être présentés en haut ou en gauche.
Pour répondre à ce besoin, on peut utiliser la classe « order » pour contrôler l’ordre des blocs.
Dans l’exemple de Monsieur Tortue et Monsieur Lapin, sur l’écran de taille large, Monsieur Lapin est devant de Monsieur Tortue. Mais sur l’écran de taille plus petite, Monsieur Tortue avance plus vite que Monsieur Lapin.

Pourquoi pas un affichage flexible ?

Est-ce que vous utilisez encore div *10 pour construire des éléments pour votre page web ? Est-ce que vous voulez sortir ce travail laborieux ?
À nos jours, nous établissons rarement le ficher de *.html page par page. Par contre, Angular est largement utilisé pour l’application responsive. Angular offre une solution, la directive ngFor, une des 4 notions importantes dans sa stucture.
Revien à la question d’affichage flexible, nous pourrions utiliser la directive ngFor. Mais attention, on ne peut pas directement mettre ngFor comme nous utilisons ngClass. Il faut combiner * avec ngFor.

Voici un exemple de notre page web :

Pour construire 9 boîtes pour le jeu Tic-Tac-Toe, nous mettons *ngFor dans un div, puis ngFor lit des éléments dans un vecteur sous forme :
let element of vecteur; index as i;
Par la suite, nous pourrions bind l’info dans notre page. Comme notre vecteur contient 9 éléments, donc 9 boîtes sont construites ainsi.

La forme ci-dessous sont basique. ngFor offre des autres attributs pour servir nos besoins. Si nous voulons identifier le premier élément, il faut simplement ajouter first as isFirst. Dans ce cas isFirst est un boolean. Même logique pour even as isEven, odd as isOdd, etc.

Pourquoi it faut combiner * avec ngFor ? Dans Angular, * est pour les directives qui pourrait changer la structure de l’affichage. ngFor ajoute ou supprime les éléments, il change la structure de la page, donc, il faut mettre * avant ngFor. Pour le ngIf, il peut cacher un élément, donc, il est utilisé aussi avec *.
